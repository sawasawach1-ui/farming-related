# 農作業・出荷記録アプリ 仕様書

作成日：2026-04-06

---

## 概要

農作業と出荷の記録をテキスト（または音声）で入力し、AIが自動解析してGoogleスプレッドシートに保存するWebアプリ。

---

## システム構成

```
ブラウザ（HTML/CSS/JS）
        ↕ fetch（HTTP GET）
Google Apps Script（GAS）
        ↕ SpreadsheetApp API
Googleスプレッドシート（データベース代わり）
```

### ファイル構成

| ファイル | 役割 |
|---------|------|
| `Index.html` | 画面の骨格（5画面タブ構成） |
| `css/style.css` | デザイン・レイアウト |
| `js/config.js` | 定数・APIキー管理・共通ユーティリティ |
| `js/api.js` | GASとの通信・オフラインキャッシュ |
| `js/analyze.js` | AI解析（Claude API）・キーワード解析 |
| `js/records.js` | データの保存・更新・削除（CRUD） |
| `js/ui.js` | 画面描画・フォーム生成 |
| `js/app.js` | 全体の司令塔・画面切り替え・起動処理 |
| `gas/Code.gs` | GASサーバー側の処理（参照用） |

---

## 画面構成

| タブ | 内容 |
|------|------|
| 入力 | テキスト・音声で作業/出荷内容を入力しAI解析 |
| 確認 | 解析結果を確認・修正してスプレッドシートに保存 |
| 未確定 | 保存済みだが未確定の記録一覧（確定・編集・削除） |
| 確定済み | 確定した記録の一覧 |
| 設定 | GAS接続テスト・Claude APIキーの設定 |

---

## データ仕様

### スプレッドシート列定義

| 列名 | 内容 |
|------|------|
| id | レコードの一意ID（自動生成） |
| type | `work`（作業）または `shipping`（出荷） |
| date | 記録日（YYYY-MM-DD） |
| person | 担当者名 |
| location | 場所（ハウス番号など） |
| content | 作業内容 |
| duration | 作業時間 |
| destination | 出荷先 |
| item | 品目 |
| quantity | 数量 |
| notes | 備考 |
| status | `pending`（未確定）または `confirmed`（確定済み） |
| createdAt | 作成日時（ISO 8601） |
| rawInput | 元の入力テキスト |

### ステータス遷移

```
入力 → [AI解析] → pending（未確定）→ confirmed（確定済み）
```

---

## AI解析

### Claude APIあり（高精度）

- モデル：`claude-haiku-4-5`
- APIキーはブラウザの `localStorage` に保存
- 自然文からtype/date/person/location/content等を自動抽出しJSON返却

### キーワード解析（APIキーなし・フォールバック）

- `js/config.js` に定義したキーワードリストでマッチング
- 出荷ワード：出荷、納品、届け、送り
- 作業ワード：収穫、作業、植え、種まき、水やり 等
- 出荷先：農総研、JA、直売、コボコボ
- 品目：きゅうり、トマト、ナス 等

## GAS（バックエンド）仕様

### エンドポイント

すべて HTTP GET で通信（file://からのPOSTはCORSでブロックされるため）。

| actionパラメータ | 処理 |
|----------------|------|
| なし | 全レコードをJSON形式で返す |
| `save` | 新規レコードを行として追加 |
| `update` | 既存レコードを更新（なければ追加） |
| `delete` | 指定IDの行を削除 |

### 接続先情報

| 項目 | 値 |
|------|-----|
| スプレッドシートID | `1-wvBlJ2CxxmvivY-nKnufa6OFNcqmDwZhdWTMGUNyqw` |
| シート名 | `記録` |
| GAS デプロイURL | `js/config.js` の `SCRIPT_URL` を参照 |

---

## オフライン対応

- 起動時にキャッシュ（`localStorage`）を即表示し、その後GASから最新データを取得
- GAS通信失敗時はキャッシュを表示し続ける
- 保存・更新・削除に失敗した場合は自動でロールバック

## セットアップ手順

### 初回セットアップ

1. Googleスプレッドシートを作成
2. スプレッドシートのメニューから **「拡張機能」→「Apps Script」** を開く
3. `gas/Code.gs` の内容をGASエディタに貼り付けて保存
4. GASエディタで `ensureSheet` を実行して権限を許可
5. **「デプロイ」→「新しいデプロイ」** でウェブアプリとしてデプロイ
   - 次のユーザーとして実行：**自分**
   - アクセスできるユーザー：**全員**
6. 発行されたURLを `js/config.js` の `SCRIPT_URL` に設定

### アプリの起動

- VS Code の **Live Server** 拡張機能で `Index.html` を開く（`http://127.0.0.1:5500/Index.html`）
- `file://` で直接開くとCORSエラーになるため不可

### GAS更新時の注意

- `Code.gs` を変更した場合は**必ず「新しいバージョン」で再デプロイ**すること
- 再デプロイするとURLが変わるので、`js/config.js` の `SCRIPT_URL` も更新すること

## トラブルシューティング

| 症状 | 原因 | 対処 |
|------|------|------|
| 「読み込み失敗」が出る | GASへの通信エラー | GASのデプロイURL・アクセス権限を確認 |
| `getSheetByName` エラー | GASとスプレッドシートが別アカウント、またはIDが間違い | スプレッドシートから直接Apps Scriptを開いて同じアカウントで管理する |
| 保存しても反映されない | GASエラーが無視されていた（修正済み） | エラーメッセージを確認してGAS設定を見直す |
| `file://` で開いている | CORSブロック | Live Serverで `http://` 経由で開く |
